# gdedit v0.2.0

## Highlights

- Editing-first terminal workflow with a more usable Control Hub.
- Slash file commands: `/open`, `/write`, `/saveas`.
- Per-file Control Hub conversation history with `F3` popup.
- Reply preview truncation in the main UI while full wrapped history remains available in the popup.
- Better Control Hub text editing: delete on selection, word movement/selection, safer Alt handling.
- Memo capture improvements:
  - explicit `->` payload support
  - `-.` shorthand for multi-line memo bullets
  - app/project/system memo scope routing
  - app memos stored at `memoRoot/app/[app-name].md`

## Release Notes Draft

gdedit v0.2.0 improves the editing-first terminal workflow with better Control Hub commands,
per-file conversation history, clearer memo routing, and safer release automation.

Major user-facing changes:

- open/save file commands directly from the Control Hub
- better text editing behavior in the Control Hub
- wrapped file-specific history popup via `F3`
- deterministic memo capture from one-line input
- app memos restored to `memoRoot/app/[app-name].md`
